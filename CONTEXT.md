# Zotero LLM Plugin

一个集成大语言模型到 Zotero 7 论文阅读工作流的插件，通过 OpenAI 兼容 API 接入第三方模型，提供即时问答、深度思考、互动教学和 Canvas 协作编辑等功能。

## Language

**会话 (Session)**:
用户与 AI 在单篇论文上下文中的完整、可恢复对话流。包含普通消息和 Canvas 块；同一篇论文在多个窗口中打开时共享同一个会话。
_Avoid_: 对话、聊天记录、线程

**窗口运行态 (Window Runtime State)**:
某个 Zotero 窗口中临时存在的交互状态，例如当前输入、展开/折叠状态和请求中的界面状态。它不属于论文记忆，也不决定会话归属。
_Avoid_: 窗口会话、本地会话、临时会话

**Canvas 块 (Canvas Block)**:
嵌入在会话流中的特殊消息类型。AI 在此生成/编辑长文本（翻译、摘要、改写、论文结构提取、公式/图表解释），用户可继续追问、编辑、一键保存。切换论文时未确认的 Canvas 自动进入草稿状态。
_Avoid_: 编辑器、文档、工作区、白板

**Canvas 草稿 (Canvas Draft)**:
Canvas 块中用户尚未确认保存的中间状态。每个草稿归属于一个具体 Canvas 块；同一会话可同时存在多个 Canvas 草稿。
_Avoid_: 未保存内容、临时编辑

**思考模式 (Thinking Mode)**:
AI 回复时展示推理链的模式。包含三种子模式：
- **CoT 推理** — 展示逐步推理过程再给出最终答案
- **深度研究** — 自动拆解问题、多轮推理、综合生成
- **批判反思** — 对论文论证给出批判性评价和替代解释
_Avoid_: 深度思考、推理模式、Reasoning

**教学模式 (Teaching Mode)**:
AI 扮演导师的模式。包含三种子模式：
- **讲解** — 用通俗语言解释选中段落
- **考测** — 苏格拉底式问答检验理解
- **推导** — 逐步引导推导论文中的公式/逻辑
_Avoid_: 学习模式、教育模式、导师模式

**上下文策略 (Context Strategy)**:
用户可配置的策略，决定每次 AI 请求携带多少论文周边上下文（仅选中文字 → 选中文字 + 段落 → 选中文字 + 整页 + 论文元数据）。
_Avoid_: 上下文范围、Prompt 范围、Token 预算

**模型后端 (LLM Backend)**:
OpenAI 兼容格式（`/v1/chat/completions`）的 API 端点。用户配置 base_url、api_key、model_name，全局唯一。
_Avoid_: API 配置、模型设置、中转站

**笔记归属 (Note Ownership)**:
用户确认 Canvas 产出后，自动将内容保存为当前论文的 Zotero Note 子条目。
_Avoid_: 保存策略、导出
